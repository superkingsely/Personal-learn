To set up Swagger UI in .NET 10 (which natively relies on the Microsoft.AspNetCore.OpenApi document generator), you need to pull in the Swashbuckle.AspNetCore.SwaggerUI NuGet package and tell the Swagger UI where to find the /openapi/v1.json endpoint generated by .NET 10's MapOpenApi. [1, 2, 3] 
Here is how you can set it up in three steps:
## Step 1: Install the Swagger UI Package
Add the Swagger UI package to your project via the terminal: [4] 

dotnet add package Swashbuckle.AspNetCore.SwaggerUI

## Step 2: Configure Program.cs
Update your Program.cs file to map both the OpenAPI document endpoint and the Swagger UI middleware. [3, 5, 6] 

using Swashbuckle.AspNetCore.SwaggerUI;
var builder = WebApplication.CreateBuilder(args);
// Registers services required for OpenAPI document generation
builder.Services.AddOpenApi();
var app = builder.Build();
// Configure the HTTP request pipeline.if (app.Environment.IsDevelopment())
{
    // Generates the OpenAPI 3.1 JSON document at /openapi/v1.json
    app.MapOpenApi();

    // Serves the Swagger UI at /swagger
    app.UseSwaggerUI(options =>
    {
        options.SwaggerEndpoint("/openapi/v1.json", "My API V1");
    });
}

app.MapGet("/", () => "Hello World!");

app.Run();

## Step 3: Run the Application
Start your app, navigate to https://localhost:<port>/swagger, and you will see the familiar Swagger UI loaded. [7, 8, 9] 
Alternatively, because native OpenAPI was enhanced in .NET 10, you can also use Scalar as an interactive, built-in alternative by installing Scalar.AspNetCore and replacing app.UseSwaggerUI with app.MapScalarApiReference(). [1, 10]
